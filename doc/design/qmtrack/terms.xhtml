• issue: A record in the qmtrack system is called an issue. We choose a general and neutral term, since issues may correspond not only to bug reports, but to feature requests, documentation errors, or other, unforseen items.
  • issue database (idb): An instance of qmtrack stores all issues in a single issue database.
  • issue id (iid): Each issue is assigned a issue id unique to the issue database. An issue id is a nonnegative 31-bit integer (the nonnegative range of a standard Python integer).
  • category: Categories are the mechanism by which issues are combined into sets. Categories are primarily an organizational tool. Each issue may belong to zero or more categories. Categories are referred to by short text strings. The set of categories is enumerated as part of the idb configuration; an issue may belong to any subset of these categories.
